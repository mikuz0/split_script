# Text Splitter

Программа для разбиения текстовых файлов на отдельные файлы по маркерам с возможностью очистки текста.

## Возможности

- Разбиение текста по маркерам (1., 2., 3. и т.д.)
- Удаление содержимого в скобках
- Удаление маркеров из текста
- Очистка лишних пробелов
- Нормализация знаков препинания
- Предпросмотр результатов
- Сохранение в формате 001_имя.txt, 002_имя.txt и т.д.

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt

Правильная последовательность обработки

    Разбиение на разделы по маркерам (происходит на уровне FileSplitter)

        Сначала нужно найти все маркеры в исходном тексте

        Выделить каждый раздел целиком, включая его маркер

        Это самый первый шаг, потому что структура разделов должна сохраниться

    Для каждого раздела по отдельности:

    Шаг 1: Удаление содержимого в скобках

        Сначала обрабатываем скобки, так как они могут содержать служебную информацию

        Важно удалить скобки вместе с их содержимым, пока структура текста еще целостна

        Скобки обрабатываются в первую очередь, потому что они могут содержать символы, которые мы потом захотим удалить

    Шаг 2: Удаление маркеров

        Удаляем маркер в начале раздела (цифру с точкой)

        Удаляем цифру в конце раздела, оставляя точку

        Это делается после удаления скобок, чтобы случайно не удалить часть содержимого

    Шаг 3: Удаление указанных символов

        Теперь, когда текст уже очищен от скобок и маркеров, можно удалять отдельные символы

        Это финальная чистка, которая не повлияет на структуру

        Здесь можно удалять сноски, специальные знаки, лишние символы

    Шаг 4: Очистка текста

        Удаление лишних пробелов

        Нормализация знаков препинания

        Удаление пробелов в начале и конце

        Это самый последний шаг для приведения текста к аккуратному виду

Почему такая последовательность правильная

    Скобки раньше символов: Скобки - это структурные элементы, их содержимое нужно удалить целиком, независимо от того, какие символы там находятся. Если сначала удалить символы внутри скобок, то скобки могут стать пустыми или некорректными.

    Маркеры раньше символов: Маркеры - это служебная информация для разбиения, их нужно удалить после разбиения, но до финальной чистки, чтобы случайно не удалить цифры, которые являются частью основного текста.

    Символы в самом конце: Это финальная "косметическая" чистка. После того как все структурные элементы удалены, можно безопасно удалять конкретные символы, не боясь нарушить логику обработки.

    Очистка текста последней: Пробелы и знаки препинания приводятся в порядок после всех удалений, чтобы не создавать лишних пробелов на месте удаленных фрагментов.

Преимущества такого подхода

    Безопасность: Никакие операции не мешают друг другу

    Предсказуемость: Результат не зависит от порядка удаления символов

    Гибкость: Пользователь может добавлять любые символы для удаления, не беспокоясь, что это нарушит работу с маркерами или скобками

    Простота отладки: Легко проверить каждый этап обработки отдельно

Такой алгоритм обеспечивает корректную обработку текста независимо от того, какие символы пользователь добавил в список для удаления.
